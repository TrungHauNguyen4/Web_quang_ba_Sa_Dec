Update the existing tourism website UI design for Sa Đéc (Đồng Tháp) by adding a complete Admin Dashboard and Content Management System (CMS).

Important:

* Keep the existing public website design unchanged
* Extend the system with admin and editor roles
* Maintain the same design style, colors, typography, and spacing as the current UI

---

User Roles:

1. Admin (full access)
2. Editor (content management only)
3. Viewer (read-only in admin dashboard)

---

Admin Dashboard (Main Layout):

* Sidebar navigation (collapsible)
* Topbar with search, notifications, and user profile
* Responsive layout (desktop + tablet)

Sidebar Menu:

* Dashboard
* Manage Destinations (Địa danh)
* Manage Cuisine (Ẩm thực)
* Manage News (Tin tức)
* Manage Services (Dịch vụ hành chính)
* Media Library
* Comments
* Users & Roles
* Settings

---

1. Dashboard Page:

* Statistics cards:

  * Total articles
  * Total destinations
  * Total users
  * Total applications
* Charts (line/bar):

  * Website traffic
  * News posts over time
* Recent activities list

---

2. Content Management Pages:

2.1 Destinations Management:

* Table layout (list of destinations)
* Columns: image, title, category, status, actions
* Actions: edit, delete, view
* "Add new destination" button
* Form page:

  * Title
  * Description (rich text editor)
  * Upload images
  * Category select
  * Location (map picker)

---

2.2 Cuisine Management:

* Similar to destinations
* Fields: name, description, image, location

---

2.3 News Management:

* Article list with filter and search
* Categories: Tin mới, Sự kiện, Thông báo
* Article editor:

  * Title
  * Content (WYSIWYG editor)
  * Thumbnail
  * Publish status (draft/published)

---

3. Administrative Services Management:

* List of services
* Manage submitted applications
* Application detail page:

  * User info
  * Status update (pending, processing, completed)
* Filters by status

---

4. Media Library:

* Grid layout for images/videos
* Upload, delete, preview
* Tag or categorize media

---

5. Comments Management:

* List of comments from users
* Approve / reject / delete actions

---

6. Users & Roles:

* User list table
* Add/edit user
* Assign roles: Admin, Editor, Viewer

---

7. Settings:

* Website info (name, contact, logo)
* Social links
* Basic configuration

---

UI Design Requirements:

* Use modern admin dashboard style
* Clean, minimal, professional look
* Use cards, tables, modals, and forms
* Maintain consistency with the public UI design
* Ensure good spacing and readability

Components to include:

* Data table
